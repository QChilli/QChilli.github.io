2.示例场景

我们希望将 PA0 引脚配置为：

  • 外部中断模式（上升沿触发），
  • 使用 上拉电阻，以防止引脚悬空。

3.逐步解析

1. 遍历引脚（while 循环部分）

  • 在代码中：

    • 作用：遍历用户指定的引脚掩码，这里 GPIO_Init->Pin = GPIO_PIN_0，即 0x0001。
    • position：每次循环处理一个引脚，当前引脚是 PA0，其位置为 0。

2. 确定当前引脚（iocurrent 和 ioposition 的计算）

  • 在代码中：

    • 作用：计算出当前引脚是否有效。
    • 对于 GPIO_PIN_0，ioposition = 0x0001，iocurrent = 0x0001，表示引脚 PA0 有效。

假设 GPIO_Init->Pin = GPIO_PIN_1（表示需要配置 PA1 0x0002），以下是 while 循环的逐步执行过程：

  1. 第一次循环：

    • position = 0
    • ioposition = (1 << 0) = 0x0001，对应第 0 位。
    • iocurrent = GPIO_Init->Pin & ioposition = 0x0002 & 0x0001 = 0x0000。
    • 因为 iocurrent = 0x0000（当前引脚无效），跳过本次循环，position++。

  2. 第二次循环：

    • position = 1

    • ioposition = (1 << 1) = 0x0002，对应第 1 位。

    • iocurrent = GPIO_Init->Pin & ioposition = 0x0002 & 0x0002 = 0x0002。

    • 因为

      （当前引脚有效），进入配置逻辑：

      • 配置模式（CRL 或 CRH 寄存器）。
      • 配置上拉或下拉（BSRR 或 BRR）。
      • 如果是外部中断模式，配置 EXTI 映射、触发模式等。

  3. 第三次循环：

    • position = 2
    • ((GPIO_Init->Pin) >> position) = 0，直接退出循环。

3. 确定引脚模式和配置（switch (GPIO_Init->Mode) 部分）

  • 在代码中：

    • 作用：

      • GPIO_MODE_IT_RISING 是中断模式，上升沿触发。

      • 根据 ：

        • 配置为上拉输入模式；
        • 设置引脚对应的上拉位（ODR 位）。

4. 判断引脚属于 CRL 还是 CRH

  • 在代码中：

    • 作用：
      • 对于引脚 PA0，属于前 8 位，因此访问 CRL 寄存器。
      • registeroffset = position << 2u，即偏移为 0x00。
  1. GPIO 模式寄存器的定义：
    • 对于输入模式，STM32 使用 MODE 和 CNF 字段来配置引脚的工作模式。
    • 在上拉和下拉模式下：
      • MODE = 00（输入模式）
      • CNF = 10（上拉/下拉模式）
    • 因此，无论是上拉还是下拉，MODE 和 CNF 的组合都是相同的，导致 config 值一致。

  2. 区分上拉和下拉的方式：

    • STM32 使用引脚的 输出数据寄存器（ODR） 来决定是上拉还是下拉：
      • ODR = 1：上拉
      • ODR = 0：下拉
    • 这意味着 上拉和下拉的本质区别在于 ODR 的设置，而非 MODE 或 CNF。

  3. GPIOx->BSRR（置位寄存器）：

    • GPIOx->BSRR 是用于将对应引脚的输出数据寄存器（ODR）的某个位 设置为 1 的寄存器。
    • 写入 GPIOx->BSRR 会使指定引脚的 ODR 位被置 1。
    • 当设置为上拉模式时，需要将 ODR 置为 1，因此写入 GPIOx->BSRR。
  4. GPIOx->BRR（复位寄存器）：
    • GPIOx->BRR 是用于将对应引脚的输出数据寄存器（ODR）的某个位 清零 的寄存器。
    • 写入 GPIOx->BRR 会使指定引脚的 ODR 位被置 0。
    • 当设置为下拉模式时，需要将 ODR 清为 0，因此写入 GPIOx->BRR。

5. 写入 MODE 和 CNF 寄存器

  • 在代码中：

    • 作用：
      • 将 PA0 配置为输入模式（MODE 为 00），浮空或上拉（CNF）。
      • 写入 GPIOx->CRL 的对应位。

6. 配置 EXTI（外部中断模式）

  • 在代码中：

    • 作用：
      • 将 EXTI0 通道映射到 GPIOA 的第 0 引脚。

7. 配置 EXTI 触发类型

    • 作用：
      • 配置 EXTI 的触发模式：
        • 上升沿触发，设置 RTSR（上升沿触发寄存器）。
        • 对于 GPIO_MODE_IT_RISING，这里只设置 RTSR。

8. 配置中断和事件屏蔽寄存器

4.巧妙之处

position（当前正在处理的引脚编号）、ioposition（当前引脚的位掩码）、iocurrent（当前引脚的实际位值）的设计利用位掩码优点：

  1. 减少循环次数：
    • 利用 ((GPIO_Init->Pin) >> position) 提前判断，确保仅对有效的引脚进行检查，避免逐一遍历所有 16 个引脚。
  2. 快速定位引脚：
    • 通过 ioposition 和 iocurrent 的配合，直接判断当前引脚是否需要配置，无需额外计算。
  3. 灵活性：
    • 支持配置单个引脚或多个引脚，例如 GPIO_PIN_0 | GPIO_PIN_1，统一逻辑下处理任意组合的引脚。

2.CAN

1.CAN_InitTypeDef

CAN_InitTypeDef 是一个用于配置 CAN 控制器的初始化结构体。

1. Prescaler

  • 含义：用于配置 CAN 时钟的分频系数，以调整时间量程（Time Quantum, TQ）的长度。

  • 范围：1 到 1024。

  • 对应寄存器：

    • 时钟分频因子决定了 CAN 位时间的基本时间量程。
    • 实际波特率公式：BaudRate = CAN_Clock / ((Prescaler) * (SyncSeg + TimeSeg1 + TimeSeg2))

2. Mode

  • 含义：指定 CAN 的操作模式。

  • 值：

    • 正常模式：数据正常传输。
    • 环回模式：仅在内部测试。
    • 静默模式：监听总线，不主动传输。
    • 静默环回模式：用于自测和诊断。

  • 对应寄存器：

    • 设置 CAN 控制器的工作模式，例如 Normal 模式、Loopback 模式等。

3. SyncJumpWidth

  • 含义：同步跳转宽度（SJW），表示 CAN 硬件在重同步时允许调整的最大时间量程（TQ）数量。

  • 范围：1 到 4。

  • 对应寄存器：

    • 在重同步时，如果出现总线上的相位误差，允许调整的最大时间范围。

4. TimeSeg1

  • 含义：时间段 1（Bit Segment 1），指定一个位时间中段 1 的时间量程数。

  • 范围：1 到 16。

  • 对应寄存器：

    • 位于同步段之后，包含采样点的设置。
    • 决定采样点的位置。

5. TimeSeg2

  • 含义：时间段 2（Bit Segment 2），指定一个位时间中段 2 的时间量程数。

  • 范围：1 到 8。

  • 对应寄存器：

    • 位于采样点之后，用于信号的相位缓冲。

6. TimeTriggeredMode

  • 含义：是否启用时间触发通信模式。

  • 值：ENABLE 或 DISABLE。

  • 对应寄存器：

    • 启用后，消息传输与时间标记结合，可以在消息中嵌入时间信息。

7. AutoBusOff

  • 含义：是否启用自动总线关闭管理。

  • 值：ENABLE 或 DISABLE。

  • 对应寄存器：

    • 当 CAN 控制器检测到总线错误后，可以自动离线并尝试恢复。

8. AutoWakeUp

  • 含义：是否启用自动唤醒功能。

  • 值：ENABLE 或 DISABLE。

  • 对应寄存器：

    • 在低功耗模式下，当接收到总线活动时自动唤醒。

9. AutoRetransmission

  • 含义：是否启用报文自动重发功能。

  • 值：ENABLE 或 DISABLE。

  • 对应寄存器：

    • 禁用后，错误帧不会自动重发，适用于只发一次的应用场景。

10. ReceiveFifoLocked

  • 含义：是否启用接收 FIFO 锁定模式。

  • 值：ENABLE 或 DISABLE。

  • 对应寄存器：

    • 锁定模式下，FIFO 满时不再覆盖旧数据。

11. TransmitFifoPriority

  • 含义：是否启用传输 FIFO 优先级模式。

  • 值：ENABLE 或 DISABLE。

  • 对应寄存器：

    • 启用后，优先根据报文 ID 的优先级决定发送顺序。

2.CAN_FilterTypeDef

CAN_FilterTypeDef 是一个用于配置 CAN 过滤器的初始化结构体。

1. FilterIdHigh

  • 含义：过滤器标识符的高 16 位。

  • 范围：0x0000 到 0xFFFF。

  • 对应寄存器：

    • 在 32 位模式下，用于存储消息标识符的高 16 位。
    • 在 16 位模式下，用于存储第一个标识符的高 16 位。

2. FilterIdLow

  • 含义：过滤器标识符的低 16 位。

  • 范围：0x0000 到 0xFFFF。

  • 对应寄存器：

    • 在 32 位模式下，用于存储消息标识符的低 16 位。
    • 在 16 位模式下，用于存储第二个标识符的低 16 位。

3. FilterMaskIdHigh

  • 含义：过滤器掩码的高 16 位。

  • 范围：0x0000 到 0xFFFF。

  • 对应寄存器：

    • 在 32 位模式下，用于存储消息掩码的高 16 位。
    • 在 16 位模式下，用于存储第一个掩码的高 16 位。

4. FilterMaskIdLow

  • 含义：过滤器掩码的低 16 位。

  • 范围：0x0000 到 0xFFFF。

  • 对应寄存器：

    • 在 32 位模式下，用于存储消息掩码的低 16 位。
    • 在 16 位模式下，用于存储第二个掩码的低 16 位。

5. FilterFIFOAssignment

  • 含义：指定符合过滤条件的消息分配到哪个 FIFO 缓冲区。

  • 范围：0 或 1。

  • 对应寄存器：

    • 指定消息分配到 FIFO 0 或 FIFO 1。

6. FilterBank

  • 含义：需要配置的滤波器编号。

  • 范围：

    • 单 CAN 实例：0 到 13。
    • 双 CAN 实例：0 到 27。

  • 对应寄存器：

    • 用于选择需要初始化的过滤器编号。

7. FilterMode

  • 含义：设置过滤器的工作模式。

  • 值：

    • 标识符掩码模式（Id/Mask）。
    • 标识符列表模式（Id List）。

  • 对应寄存器：

    • 配置过滤器为掩码模式或列表模式。

8. FilterScale

  • 含义：定义过滤器的位宽。

  • 值：

    • 0：16 位模式。
    • 1：32 位模式。

  • 对应寄存器：

    • 配置过滤器的位宽模式。

9. FilterActivation

  • 含义：启用或禁用过滤器。

  • 值：

    • ENABLE：启用过滤器。
    • DISABLE：禁用过滤器。

  • 对应寄存器：

    • 启用或禁用指定编号的过滤器。

10. SlaveStartFilterBank

  • 含义：设置从 CAN 实例的起始滤波器编号。

  • 范围：

    • 单 CAN 实例：无意义。
    • 双 CAN 实例：0 到 27。

  • 对应寄存器：

    • 配置从 CAN 实例的起始滤波器编号，低编号分配给主 CAN，高编号分配给从 CAN。
