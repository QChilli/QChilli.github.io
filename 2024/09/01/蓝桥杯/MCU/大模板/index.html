3.问题

  • 为什么 SCK = 0 必须要在 RST = 0 与 RST = 1 之间，不能在RST = 0 与 RST = 1 后面？

    ​ 在复位过程中，它的内部状态和寄存器会被重置，时钟信号也应该被禁用（SCK = 0），因此你需要确保时钟信号（SCK）

    ​ 处于低电平，以便稳定地复位 DS1302 。如果在复位信号（RST = 1）之后再设置 SCK = 0，会打乱时序和同步，导致后

    ​ 续操作（如数据传输）失败。

  • 为什么读取逻辑写完后还要操作 SCK 和 SDA ？

    从数码管读取现象来看，不操作 SCK 和 SDA大概率导致某些数据读取不准：

    SCK = 0; _nop_(); SCK = 1; _nop_();

    • ​ SCK = 0： SCK 拉低，保证时钟线处于初始化状态（根据时序，SCK = 0 -> 读数据 -> SCK = 1，图上D7之后是SCK = 0），避免任何非预期的信号波动，确保开始时刻的同步。
    • ​ SCK = 1：时钟信号的拉高是准备进行下一步操作的必要条件。它将开始提供时钟脉冲，驱动数据写入（下一步写入数据时，SCK应为高电平）。

    SDA = 0; _nop_(); SDA = 1; _nop_();

    • SDA = 0：将数据线 SDA 拉低，准备传输数据。数据线通常会在传输前先设置为低电平，确保没有其他信号干扰。
    • SDA = 1：将数据线拉高，为接下来开始数据传输做准备。数据线在高电平时表示可以开始传输有效数据。

3.问题

  • 为什么发送数据时不需要手动释放 SDA 线？

    控制权始终在主机：发送数据时，主机作为数据传输的主动方，全程控制 SDA 线的电平（逐位输出数据）。主机在发送数据时，SDA 线由主机驱动（拉高或拉低），无需释放。

    上拉电阻作用：数据发送完成后，主机停止驱动 SDA 线，此时 SDA 线通过外部上拉电阻自动恢复到高电平（空闲状态）。因此，主机无需显式释放 SDA 线。

  • 为什么接收数据时需要手动释放 SDA 线？

    控制权转移：接收数据时，SDA 线需由从机驱动。主机必须主动释放 SDA 线（停止驱动），否则从机无法修改 SDA 线的电平（总线会被主机强制拉高或拉低）。

    同步读取数据：主机通过时钟（SCL）同步读取SDA线的状态，释放SDA线后，从机在SCL的每个上升沿前设置SDA电平（传输数据位）。

  • 为什么发送应答（ACK）时需要释放 SDA 线？

    发送应答 ACK ：主机接收完数据并且发送应答后从机需要继续发送数据，主机必须释放 SDA 线，允许从机控制SDA线传输下一字节。

    发送非应答 NACK ：主机接收完数据并且发送应答后没必要显示释放 SDA 线，主机直接停止通信就好。

  • 为什么接收应答时需要释放 SDA 线？

    从机控制应答信号：接收应答时，主机需读取 SDA 线的电平（ ACK / NACK），而 SDA 线的状态由从机设置。主机必须释放 SDA线，允许从机修改其电平。

    协议时序要求：在应答周期内，主机拉高 SCL 后，从机在 SCL 高电平期间拉低 SDA（发送 ACK ）。若主机未释放 SDA 线，从机无法完成此操作，导致通信失败。
