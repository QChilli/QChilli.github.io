为什么 config 值相同？

  1. GPIO 模式寄存器的定义：
    • 对于输入模式，STM32 使用 MODE 和 CNF 字段来配置引脚的工作模式。
    • 在上拉和下拉模式下：
      • MODE = 00（输入模式）
      • CNF = 10（上拉/下拉模式）
    • 因此，无论是上拉还是下拉，MODE 和 CNF 的组合都是相同的，导致 config 值一致。
  2. 区分上拉和下拉的方式：
    • STM32 使用引脚的 输出数据寄存器（ODR） 来决定是上拉还是下拉：
      • ODR = 1：上拉
      • ODR = 0：下拉
    • 这意味着 上拉和下拉的本质区别在于 ODR 的设置，而非 MODE 或 CNF。

  1. GPIOx->BSRR（置位寄存器）：
    • GPIOx->BSRR 是用于将对应引脚的输出数据寄存器（ODR）的某个位 设置为 1 的寄存器。
    • 写入 GPIOx->BSRR 会使指定引脚的 ODR 位被置 1。
    • 当设置为上拉模式时，需要将 ODR 置为 1，因此写入 GPIOx->BSRR。
  2. GPIOx->BRR（复位寄存器）：
    • GPIOx->BRR 是用于将对应引脚的输出数据寄存器（ODR）的某个位 清零 的寄存器。
    • 写入 GPIOx->BRR 会使指定引脚的 ODR 位被置 0。
    • 当设置为下拉模式时，需要将 ODR 清为 0，因此写入 GPIOx->BRR。

4.巧妙之处

position（当前正在处理的引脚编号）、ioposition（当前引脚的位掩码）、iocurrent（当前引脚的实际位值）的设计利用位掩码优点：

  1. 减少循环次数：
    • 利用 ((GPIO_Init->Pin) >> position) 提前判断，确保仅对有效的引脚进行检查，避免逐一遍历所有 16 个引脚。
  2. 快速定位引脚：
    • 通过 ioposition 和 iocurrent 的配合，直接判断当前引脚是否需要配置，无需额外计算。
  3. 灵活性：
    • 支持配置单个引脚或多个引脚，例如 GPIO_PIN_0 | GPIO_PIN_1，统一逻辑下处理任意组合的引脚。

位掩码解释

位掩码（Bitmask） 是一种通过按位操作（位与、位或等）来选择或操作二进制数据特定位的方法，主要用于高效地处理特定位的操作，比如启用、禁用、检测或切换特定位。

位掩码的基本概念

位掩码本质上是一个二进制数，通过其每一位的 0 或 1 来控制另一个数的特定位是否参与操作。

  • “掩码” 的意思是“遮盖”或“过滤”，通过掩码可以过滤出需要的特定位。
  • 常见操作包括 提取、设置、清除 和 切换（翻转）。

为什么需要位掩码

位掩码用于高效地操作数据的特定位，尤其是在嵌入式编程中经常用到。例如：

  1. 检测特定位是否为 1（如 GPIO 引脚状态）。
  2. 设置某个位为 1（如启用某个硬件功能）。
  3. 清除某个位为 0（如禁用某个硬件功能）。
  4. 切换某个位的值（如翻转某个引脚的状态）。

这些操作可以通过按位操作（位与 &、位或 |、按位取反 ~ 等）实现。

实际意义

假设 GPIO_Init->Pin = GPIO_PIN_1 | GPIO_PIN_3 = 0b00001010（同时配置 PA1 和 PA3）：

  • position = 0 时，ioposition = 0b00000001，iocurrent = 0b00001010 & 0b00000001 = 0b00000000（跳过）。
  • position = 1 时，ioposition = 0b00000010，iocurrent = 0b00001010 & 0b00000010 = 0b00000010（配置 PA1）。
  • position = 2 时，ioposition = 0b00000100，iocurrent = 0b00001010 & 0b00000100 = 0b00000000（跳过）。
  • position = 3 时，ioposition = 0b00001000，iocurrent = 0b00001010 & 0b00001000 = 0b00001000（配置 PA3）。

通过这样的位掩码操作，只需要一整个 while 循环的完整运行过程，即包括多次迭代，直到所有引脚都被检查完,即可有效配置多个 GPIO 引脚。

总结

  • 位掩码 是一种高效的位操作工具，用于控制或检测二进制数据的特定位。
  • 在嵌入式编程中，位掩码常用于 GPIO 配置、寄存器操作等场景。
  • 在 HAL_GPIO_Init 中，位掩码被用来逐个检查用户指定的引脚，避免不必要的全引脚遍历，提升效率。
